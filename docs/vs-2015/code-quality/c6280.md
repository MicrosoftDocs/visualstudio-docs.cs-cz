# <a name="c6280"></a>C6280
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C6280: \<variable> je přiděleno pomocí \<function> , ale odstraněno s \<function>  
  
 Toto upozornění indikuje, že volající funkce má nekonzistentně přidělenou paměť pomocí funkce z jedné rodiny přidělení paměti a uvolnila ji pomocí funkce z jiné rodiny přidělování paměti. Analyzátor kontroluje tuto podmínku pouze v případě, že `_Analysis_mode_(_Analysis_local_leak_checks_)` je zadána anotace SAL. Ve výchozím nastavení je tato poznámka určena pro kód režimu jádra systému Windows (ovladač). Další informace o poznámkách SAL najdete v tématu [Použití poznámek SAL k omezení vad kódu C/C++](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md).  
  
 Toto upozornění by například bylo vytvořeno, pokud je paměť přidělena pomocí, `malloc` ale uvolněna pomocí `GlobalFree` nebo `delete` . V určitých případech neshody mezi polem `new[]` a skalárním `delete` jsou popsána přesnější upozornění místo tohoto.  
  
## <a name="example"></a>Příklad  
 Následující vzorový kód vygeneruje toto upozornění.  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <stdlib.h>  
  
_Analysis_mode_(_Analysis_local_leak_checks_)  
  
void f(int arraySize)  
{  
    int *pInt = (int *)calloc(arraySize, sizeof (int));  
    // code ...  
    delete pInt;  
}  
  
```  
  
 Chcete-li toto upozornění opravit, použijte tento kód:  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <stdlib.h>  
  
_Analysis_mode_(_Analysis_local_leak_checks_)  
  
void f(int arraySize)  
{  
    int *pInt = (int *)calloc(arraySize, sizeof (int));  
    // code ...  
    free(pInt);  
}  
  
```  
  
 Různé definice rozhraní API mohou používat různé haldy. Například `GlobalAlloc` používá systémovou haldu a `free` používá haldu procesu. To může způsobit poškození a zhroucení paměti.  
  
 Tyto nekonzistence se vztahují na `new` / `delete` `malloc` / `free` mechanismy přidělování paměti a. Chcete-li se vyhnout těmto druhům potenciálních nekonzistencí, použijte mechanismy, které jsou k dispozici v knihovně C++ Standard Template Library (STL). Mezi ně patří [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)a [Vector](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Další informace naleznete v tématu [inteligentní ukazatele](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) a [Standardní knihovna C++](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <vector>  
#include <memory>  
  
using namespace std;  
  
_Analysis_mode_(_Analysis_local_leak_checks_)  
  
void f(int arraySize)  
{  
    // use unique_ptr instead of calloc/malloc/new  
    unique_ptr<int[]> pInt(new int[arraySize]);  
  
    // code ...  
  
    // No need for free because unique_ptr   
    // cleans up when out of scope.  
}  
  
```
