# <a name="c6250"></a>C6250
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C6250: volání funkce \<> VirtualFree bez příznaku MEM_RELEASE může uvolnit paměť, ale ne popisovače adresy (vad); výsledky při nevracení adresního prostoru  
  
 Toto upozornění signalizuje, že volání `VirtualFree` bez příznaku `MEM_RELEASE` pouze zruší stránky a neuvolní je. Chcete-li zrušit potvrzení a vydávat stránky, použijte při volání `VirtualFree`příznak `MEM_RELEASE`. Pokud jsou všechny stránky v oblasti potvrzeny, funkce nejprve zruší potvrzení a pak je uvolní. Po této operaci jsou stránky ve stavu Free. Pokud zadáte tento příznak, `dwSize` musí být nula a `lpAddress` musí odkazovat na základní adresu vrácenou funkcí `VirtualAlloc` při rezervaci oblasti. Tato funkce se nezdařila, pokud není splněna jedna z těchto podmínek.  
  
 Toto upozornění můžete ignorovat, pokud kód později uvolní adresní prostor voláním `VirtualFree` s příznakem `MEM_RELEASE`.  
  
 Další informace najdete v tématu [VirtualAlloc](https://msdn.microsoft.com/library/aa366887.aspx) a [VirtualFree](https://msdn.microsoft.com/library/aa366892.aspx).  
  
 Použití VirtualAlloc a VirtualFree má mnoho nástrah z pohledu nevracení paměti a výjimek. Chcete-li se vyhnout těmto typům nevrácených chyb a potížím s výjimkami, použijte C++ mechanismy, které jsou k dispozici pomocí knihovny Standard Template Library (STL). Mezi ně patří [shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860), [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)a [Vector](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f). Další informace najdete v tématu [inteligentní ukazatele](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435) a [ C++ standardní knihovna](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee).  
  
## <a name="example"></a>Příklad  
 Následující vzorový kód vygeneruje toto upozornění:  
  
```  
#include <windows.h>  
#include <stdio.h>  
#define PAGELIMIT 80              
  
DWORD dwPages = 0;  // count of pages   
DWORD dwPageSize;   // page size   
  
VOID f( )  
{  
  LPVOID lpvBase;            // base address of the test memory  
  SYSTEM_INFO sSysInfo;      // system information  
  
  GetSystemInfo(&sSysInfo);    
  dwPageSize = sSysInfo.dwPageSize;  
  
  // Reserve pages in the process's virtual address space  
  lpvBase = VirtualAlloc(  
                         NULL,                // system selects address  
                         PAGELIMIT*dwPageSize, // size of allocation  
                         MEM_RESERVE,          
                         PAGE_NOACCESS);       
  //  
  // code to access memory   
  // ...  
  
  if (lpvBase != NULL)  
  {  
    if (VirtualFree( lpvBase, 0, MEM_DECOMMIT )) // decommit pages  
      {  
        puts ("MEM_DECOMMIT Succeeded");  
      }  
      else  
      {  
       puts("MEM_DECOMMIT failed");  
      }  
  }  
  else  
  {  
    puts("lpvBase == NULL");  
  }  
}  
```  
  
 Chcete-li toto upozornění opravit, použijte následující vzorový kód:
