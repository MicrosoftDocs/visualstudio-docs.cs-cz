# <a name="c6226"></a>C6226
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]
Upozornění C6226: implicitní přetypování mezi sémanticky odlišnými typy celých čísel: přiřazení-1 k HRESULT. Místo toho zvažte použití E_FAIL.

Toto upozornění znamená, že `HRESULT` je přiřazena nebo inicializována explicitní hodnota-1. Toto upozornění je často způsobeno náhodnou záměnou celých čísel a `HRESULT` typů. K indikaci úspěchu použijte místo toho symbolickou konstantu `S_OK` . Pro indikaci selhání použijte symbolické konstanty, které začínají E_*konstantou*, například `E_FAIL` .

Další informace najdete v tématu [úspěšná](/windows/desktop/api/winerror/nf-winerror-succeeded) a [neúspěšná](/windows/desktop/api/winerror/nf-winerror-failed) makra.

## <a name="example"></a>Příklad

Následující kód vygeneruje toto upozornění:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = -1;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Chcete-li toto upozornění opravit, použijte následující kód:

```cpp
#include <windows.h>

VOID f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  if (FAILED(CoGetMalloc(1, &pMalloc)))
  {
    hr = E_FAIL;
    // code ...
  }
  else
  {
    // code ...
  }
}
```

Pro toto upozornění `SCODE` je typ považován za `HRESULT` .

Použití `malloc` a `free` (a souvisejících rozhraní API dynamické paměti) má mnoho nástrah jako příčinu nevracení paměti a výjimek. Chcete-li se vyhnout těmto druhům úniku a potížím s výjimkami, použijte ukazatele a třídy kontejneru poskytované standardní knihovnou jazyka C++. Mezi ně patří [shared_ptr](/cpp/standard-library/shared-ptr-class), [unique_ptr](/cpp/standard-library/unique-ptr-class)a [Vector](/cpp/standard-library/vector). Další informace naleznete v tématu [inteligentní ukazatele](/cpp/cpp/smart-pointers-modern-cpp) a [Standardní knihovna C++](/cpp/standard-library/cpp-standard-library-reference).
