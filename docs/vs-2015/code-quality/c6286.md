# <a name="c6286"></a>C6286
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C6286: ( \<non-zero constant> &#124;&#124; \<expression> ) je vždy nenulová konstanta. \<expression> Nikdy se nevyhodnocuje a může mít vedlejší účinky  
  
 Toto upozornění znamená, že na levé straně logického operátoru nebo operace, která se vyskytuje v kontextu testu, byla zjištěna nenulová konstanta. Výsledný výraz se vždycky vyhodnotí jako TRUE. Kromě toho se zdá, že pravá strana výrazu má vedlejší účinky a dojde ke ztrátě.  
  
 Toto upozornění znamená, že možná budete chtít pečlivě zkontrolovat pravou stranu výrazu, aby se zajistilo, že všechny vedlejší účinky potřebné ke správné funkci nebudou ztraceny.  
  
 `(!0 || <expression>`Konstrukce) se běžně používá k vynucení provedení kontrolovaného bloku.  
  
## <a name="example"></a>Příklad  
 Následující kód vygeneruje toto upozornění:  
  
```  
#include <stdlib.h>  
#include <stdio.h>  
#define INPUT_TYPE 1   
  
int test();  
  
void f()  
{  
  if (INPUT_TYPE || test())   
  {  
    puts("INPUT_TYPE == 1, expression not evaluated");  
    // code...  
  }  
  else  
  {  
    puts("INPUT_TYPE == 0. Call to test() returned 0");  
    // code...  
  }  
}  
```  
  
 Následující kód ukazuje jedno možné řešení podle přerušení `if` příkazu do dvou samostatných částí:  
  
```  
#include <stdlib.h>  
#include <stdio.h>  
#define INPUT_TYPE 1   
  
int test();  
  
void f()  
{  
  int i;  
  if (INPUT_TYPE)   
  {  
    i = test();  
    // code...  
  }  
  else  
  {  
    puts("INPUT_TYPE false");  
    // code...  
  }  
}  
```
