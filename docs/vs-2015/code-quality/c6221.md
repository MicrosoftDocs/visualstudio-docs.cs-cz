# <a name="c6221"></a>C6221
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Warning C6221: implicitní přetypování mezi sémanticky odlišnými typy celých čísel: porovnává HRESULT s celým číslem. Zvažte místo toho použití makra SUCCEEDED nebo FAILed.  
  
 Toto upozornění znamená, že `HRESULT` je porovnána s celým číslem jiným než nula. Úspěch v `HRESULT` ( `S_OK` ) je reprezentován hodnotou 0. Implicitní přetypování `HRESULT` na celé číslo proto vygeneruje nesprávnou hodnotu a může vést k nesprávnému výsledku. Často je to způsobeno tím, že omylem očekáváte, že funkce vrátí celé číslo, pokud ve skutečnosti vrátí `HRESULT` .  
  
## <a name="example"></a>Příklad  
 Následující kód vygeneruje toto upozornění porovnáním `HRESULT` s celočíselnou hodnotou:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (hr == 4)  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Chcete-li toto upozornění opravit, následující kód používá `FAILED` makro:  
  
```  
#include <windows.h>  
  
HRESULT f( )  
{  
  HRESULT hr;  
  LPMALLOC pMalloc;  
  
  hr = CoGetMalloc(1, &pMalloc);  
  if (FAILED(hr))  
  {  
    // failure code ...  
    return S_FALSE;  
  }  
  else  
  {  
    // success code ...  
    return S_OK;  
  }  
}  
```  
  
 Pro toto upozornění je `SCODE` typ ekvivalentní k `HRESULT` .
