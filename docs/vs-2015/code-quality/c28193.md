# <a name="c28193"></a>C28193
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C28193: proměnná obsahuje hodnotu, kterou je třeba prozkoumat.  
  
 Toto upozornění signalizuje, že volání funkce neověřuje hodnotu zadané proměnné, která byla poskytnuta funkcí. Vrácená hodnota je popsána v poznámce s poznámkou `_Check_return_` , ale funkce volání buď nepoužívá hodnotu, nebo Přepisuje hodnotu bez přezkoumání.  
  
 Toto upozornění je podobné upozornění [C6031](../code-quality/c6031.md), ale je nahlášeno pouze v případě, že kód netestuje nebo nekontroluje hodnotu proměnné, jako je například při použití v porovnání. Pouhým přiřazením hodnoty se nepovažuje být dostatečná kontrola, aby se předešlo tomuto upozornění. Vytvoření aliasu výsledku mimo funkci je považováno za dostatečné prověřování, ale samotný výsledek by měl být opatřen poznámkami `_Check_return_` .  
  
 Některé funkce (například `strlen` ) Existují téměř výhradně pro svou návratovou hodnotu, takže jsou vhodné pro ně podávat `_Check_return_` poznámky. Pro tyto funkce může nástroj pro analýzu kódu ohlásit toto upozornění, když se nepoužívá návratová hodnota. To obvykle znamená, že kód je nesprávný, například může obsahovat reziduální kód, který může být odstraněn. V některých vzácných instancích se však návratová hodnota úmyslně nepoužívá. Nejběžnějšími z těchto instancí je, kde je vrácena délka řetězce, ale není ve skutečnosti použita před provedením jiného testu. Tento jiný test způsobí, že se má simulovat cesta, kde délka řetězce skončí nepoužitým. Pokud k tomu dojde, může být kód správný, ale může být neefektivní.  
  
 Existují dvě primární strategie pro práci s těmito případy, kdy se nepoužívá návratová hodnota:  
  
 Přeuspořádat kód tak, aby délka řetězce byla vrácena pouze podél cesty, kde je to nutné.  
  
 Použití `#pragma` Upozornění k potlačení upozornění – pokud při opětovném uspořádání kódu změníte pořadí, kód by byl příliš složitý nebo jinak méně užitečný.  
  
## <a name="example"></a>Příklad  
 Následující příklad kódu vygeneruje toto upozornění:  
  
```  
IoGetDmaAdapter(pPDO, &DevDesc, &nMapRegs);  
...  
```  
  
 Následující příklad kódu předchází tomuto upozornění:  
  
```  
IoGetDmaAdapter(pPDO, &DevDesc, &nMapRegs);  
...  
if (nMapRegs < MIN_REQUIRED_MAPS) {  
...  
}  
```
