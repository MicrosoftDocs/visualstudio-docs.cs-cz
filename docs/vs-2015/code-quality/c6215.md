# <a name="c6215"></a>C6215
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

Upozornění C6215: přetypování mezi sémanticky odlišnými typy celých čísel: typ Boolean na HRESULT  
  
 Toto upozornění indikuje, že logická hodnota je převedena na `HRESULT` . Logické typy označují úspěch nenulovou hodnotou, zatímco úspěch ( `S_OK` ) v `HRESULT` je označen hodnotou 0. Přetypování typu Boolean na typ `HRESULT` a jeho použití v testovacím výrazu vrátí nesprávný výsledek.  
  
 K tomuto upozornění dochází často, pokud je logická hodnota použita jako argument `SUCCEEDED` nebo `FAILED` makro, které explicitně přetypování jejich argumentů na `HRESULT` .  
  
## <a name="example"></a>Příklad  
 Následující kód vygeneruje toto upozornění:  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
void f( REFGUID riid1, REFGUID riid2 )  
{  
  if (SUCCEEDED( IsEqual( riid1, riid2 ) )) //warning 6215  
  {  
    // success code ...  
  }  
  else  
  {  
    // failure code ...  
  }  
}  
```  
  
 Obecně platí, `SUCCEEDED` že `FAILED` makra nebo by měla být použita pouze na `HRESULT` .  
  
 Chcete-li toto upozornění opravit, použijte následující kód:  
  
```  
#include <windows.h>  
BOOL IsEqual(REFGUID, REFGUID);  
  
void f( REFGUID riid1, REFGUID riid2 )  
{  
  if (IsEqual( riid1, riid2 ) == TRUE)   
  {  
    // code for riid1 == riid2  
  }  
  else  
  {  
    // code for riid1 != riid2  
  }  
}  
```  
  
 Další informace najdete v tématu [úspěšné makro](https://msdn.microsoft.com/library/ms687197.aspx) a [neúspěšné makro](https://msdn.microsoft.com/library/ms693474.aspx) .
