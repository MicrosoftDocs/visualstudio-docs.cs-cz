# <a name="ca1833-use-asspan-or-asmemory-instead-of-range-based-indexers-for-getting-span-or-memory-portion-of-an-array"></a>CA1833: Pro získání části Span nebo Memory pole používat místo indexerů založených na rozsahu metodu AsSpan nebo AsMemory

|Položka|Hodnota|
|-|-|
|CheckId|CA1833|
|Kategorie|Microsoft. Performance|
|Zásadní změna|Nenarušující|

## <a name="cause"></a>Příčina

Při použití rozsahu-indexeru na poli a implicitně přiřadíte hodnotu <xref:System.Span%601> nebo <xref:System.Memory%601> .

## <a name="rule-description"></a>Popis pravidla

Použití rozsahu indexeru v poli a přiřazení k paměti nebo typu rozsahu: indexovací člen <xref:System.Span%601> je operace, která není kopírována <xref:System.Span%601.Slice%2A?#System_Span_1_Slice_System_Int32_System_Int32_> , ale pro indexer rozsahu na Array se metoda <xref:System.Runtime.CompilerServices.RuntimeHelpers.GetSubArray%2A> použije místo <xref:System.Span%601.Slice%2A?#System_Span_1_Slice_System_Int32_System_Int32_> , která vytvoří kopii požadované části pole. Tato kopie je obvykle zbytečná, pokud se implicitně používá jako <xref:System.Span%601> hodnota nebo <xref:System.Memory%601> . Pokud není kopie zamýšlená, použijte <xref:System.MemoryExtensions.AsSpan%2A?#System_MemoryExtensions_AsSpan__1___0___> metodu nebo, <xref:System.MemoryExtensions.AsMemory%2A?#System_MemoryExtensions_AsMemory__1___0___> abyste se vyhnuli zbytečnému kopírování. Pokud je kopie určena, buď ji přiřaďte k místní proměnné, nebo přidejte explicitní přetypování. Analyzátor generuje pouze v případě, že je pro výsledek operace indexeru rozsahu použit implicitní přetypování.

### <a name="detects"></a>Zjistit

Implicitní převody:
- `Span<SomeT> slice = arr[a..b];`
- `Memory<SomeT> slice = arr[a..b];`

### <a name="does-not-detect"></a>Nedetekuje

Explicitní převody:
- `Span<SomeT> slice = (Span<SomeT>)arr[a..b];`
- `Memory<SomeT> slice = (Memory<SomeT>)arr[a..b];`

## <a name="how-to-fix-violations"></a>Jak opravit porušení

Chcete-li opravit porušení tohoto pravidla: použijte <xref:System.MemoryExtensions.AsSpan%2A?#System_MemoryExtensions_AsSpan__1___0___> <xref:System.MemoryExtensions.AsMemory%2A?#System_MemoryExtensions_AsMemory__1___0___> metodu rozšíření nebo, aby nedocházelo k vytváření zbytečných kopií dat.

```csharp
class C
{
    public void TestMethod(byte[] arr)
    {
        // The violation occurs for both statements below
        Span<byte> tmp2 = arr[0..5];
        Memory<byte> tmp4 = arr[5..10];     
        ...
    }
}
```

```csharp
class C
{
    public void TestMethod(byte[] arr)
    {
        // The violations fixed with AsSpan or AsMemory accordingly
        Span<byte> tmp2 = arr.AsSpan()[0..5];
        Memory<byte> tmp4 = arr.AsMemory()[5..10];  
        ...
    }
}
```

> Pro toto pravidlo v aplikaci Visual Studio je k dispozici oprava kódu. Pokud ho chcete použít, umístěte kurzor na porušení pole a stiskněte **klávesu CTRL** + **.** (tečka). V seznamu zobrazených možností **použijte místo indexeru založeného na rozsahu možnost použít AsMemory** .
>

## <a name="when-to-suppress-warnings"></a>Kdy potlačit upozornění

Pokud je vytvoření kopie zamýšlené, je bezpečné potlačit porušení tohoto pravidla. Chcete-li toto upozornění potlačit, můžete jednoduše přidat explicitní přetypování.

```csharp
class C
{
    public void TestMethod(byte arr[])
    {
        // The violation occurs
        Span<byte> tmp1 = arr[0..5];
        Memory<byte> tmp2 = arr[5..10]; 
        ...
    }
}
```

```csharp
class C
{
    public void TestMethod(byte arr[])
    {
        // The violation fixed with explicit casting
        Span<byte> tmp1 = (Span<byte>)arr[0..5];
        Memory<byte> tmp2 = (Memory<byte>)arr[5..10];
        ...
    }
}
```
