# <a name="c6220"></a>C6220
Upozornění C6220 – implicitní přetypování mezi sémanticky odlišnými typy celých čísel: porovnává HRESULT s-1. Zvažte místo toho použití makra SUCCEEDED nebo FAILed.

 Toto upozornění indikuje, že `HRESULT` se porovnává s explicitní hodnotou, která není `HRESULT`, což není ve správném formátu `HRESULT`. Selhání v `HRESULT` (`E_FAIL`) není reprezentované hodnotou-1. Implicitní přetypování `HRESULT` na celé číslo proto vygeneruje nesprávnou hodnotu a může vést k nesprávnému výsledku.

## <a name="example"></a>Příklad
 Ve většině případů toto upozornění je způsobeno tím, že kód omylem očekává, že funkce, která by měla vracet `HRESULT`, místo toho vrátí celé číslo, a to pomocí-1 jako hodnoty selhání. Následující ukázka kódu generuje toto upozornění:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (hr == -1)
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Je nejvhodnější použít makro `SUCCEEDED` nebo `FAILED` k otestování hodnoty `HRESULT`. Chcete-li toto upozornění opravit, použijte následující kód:

```
#include <windows.h>

HRESULT f( )
{
  HRESULT hr;
  LPMALLOC pMalloc;

  hr = CoGetMalloc(1, &pMalloc);
  if (FAILED(hr))
  {
    // failure code ...
    return E_FAIL;
  }
  else
  {
    // success code ...
    return S_OK;
  }
}
```

 Pro toto upozornění je typ `SCODE` ekvivalentem `HRESULT`.

 Explicitní porovnání je vhodné ke kontrole konkrétních hodnot `HRESULT`, například `E_FAIL`. V opačném případě použijte makra `SUCCEEDED` nebo `FAILED`.

 Další informace najdete v tématu [úspěšné makro](/windows/win32/api/winerror/nf-winerror-succeeded) a [makro, které selhalo](/windows/win32/api/winerror/nf-winerror-failed).
