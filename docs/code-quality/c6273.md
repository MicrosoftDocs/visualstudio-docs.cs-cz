# <a name="c6273"></a>C6273
Upozornění 6273 – hodnota bez celého čísla předaná jako parametr \<number >, pokud je při volání metody @no__t-> 1function vyžadováno celé číslo: Pokud je předávána hodnota ukazatele, měla by se použít% p.

 Toto upozornění označuje, že formátovací řetězec Určuje celé číslo, například specifikace `%d`, délka nebo priorita pro `printf`, ale jiné než celé číslo, jako je například `float`, řetězec nebo `struct`, se předává jako parametr. Tato vada může mít za následek nesprávný výstup.

## <a name="example"></a>Příklad
 Následující kód vygeneruje toto upozornění, protože místo `float` do `sprintf`function je vyžadováno celé číslo.

```
#include <stdio.h>
#include <string.h>

void f_defective()
{
  char buff[50];
  float f=1.5;

  sprintf(buff, "%d",f);
}
```

 Následující kód používá přetypování Integer k opravě tohoto upozornění:

```
#include <stdio.h>
#include <string.h>

void f_corrected()
{
  char buff[50];
  float f=1.5;

  sprintf(buff,"%d",(int)f);
}
```

 Následující kód používá bezpečnou funkci manipulace s řetězci `sprintf_s` pro opravu tohoto upozornění:

```
#include <stdio.h>
#include <string.h>

void f_safe()
{
  char buff[50];
  float f=1.5;

  sprintf_s(buff,50,"%d",(int)f);
}
```

 Toto upozornění se nevztahuje na systémy Windows 9x a Windows NT verze 4, protože% p není na těchto platformách podporováno.
