# <a name="c6284"></a>C6284
Warning C6284: objekt byl předán jako parametr% d, pokud je při volání > \<function vyžadován řetězec.

 Toto upozornění indikuje, že formátovací řetězec Určuje řetězec, například specifikace `%s` pro `printf` nebo `scanf`, ale místo toho byl předán C++ objekt.

 Tato vada může způsobit nesprávný výstup nebo selhání.

 Tato zpráva je často hlášena z důvodu předání C++ objektu implementující nějaký typ řetězce, například `std::string`, `CComBSTR` nebo `bstr_t`, do volání jazyka C @no__t -4-Style. V závislosti na implementaci C++ třídy, to znamená, pokud jsou definovány správné operátory přetypování, lze řetězcové C++ objekty často použít transparentně pokaždé, když jsou vyžadovány řetězce jazyka C; vzhledem k tomu, že parametry @no__t funkce ve stylu -2 jsou v podstatě netypové, není k dispozici žádný převod na řetězec.

 V závislosti na objektu může být vhodné vložit operátor `static_cast` do příslušného typu řetězce, například `char *` nebo `TCHAR``*` nebo volat členskou funkci, která vrací řetězec, například `c_str()`, na instancích `std::string`.

## <a name="example"></a>Příklad
 Následující kód vygeneruje toto upozornění, protože funkce `sprintf` předává `CComBSTR`:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf(buff,"%ws",bstrValue);
}
```

 Následující kód používá statické přetypování k opravě tohoto upozornění:

```
#include <atlbase.h>
#include <stdlib.h>

void f()
{
  char buff[50];
  CComBSTR bstrValue("Bye");

  sprintf_s(buff,50,"%ws",static_cast<wchar_t *>(bstrValue));
}
```
